# 博客系统功能设计

## 1. 系统概述

博客系统是一个支持多用户的内容管理平台，提供文章发布、分类管理、标签管理、评论互动等核心功能。

## 2. 功能模块设计

### 2.1 用户管理模块

#### 2.1.1 用户注册
- 支持邮箱注册
- 用户名唯一性校验
- 密码加密存储（BCrypt）
- 发送验证邮件（可选）

#### 2.1.2 用户登录
- 邮箱/用户名 + 密码登录
- JWT Token 认证机制
- 登录失败次数限制
- 记住我功能

#### 2.1.3 用户信息管理
- 查看个人资料
- 修改头像
- 修改昵称/个人简介
- 修改密码
- 绑定/解绑邮箱

#### 2.1.4 用户权限
- 普通用户：发布文章、评论、管理自己的内容
- 管理员：管理所有内容、用户管理、系统配置

### 2.2 文章管理模块

#### 2.2.1 文章发布
- 富文本编辑器（Markdown 支持）
- 文章标题、内容编辑
- 上传封面图片
- 选择分类
- 添加标签（支持多标签）
- 保存草稿 / 立即发布

#### 2.2.2 文章编辑
- 修改已发布文章
- 版本历史记录（可选）
- 文章状态管理（草稿/已发布/已下架）

#### 2.2.3 文章删除
- 软删除机制
- 删除确认提示
- 关联数据清理（评论等）

#### 2.2.4 文章查询
- 按分类筛选
- 按标签筛选
- 按作者筛选
- 按关键词搜索（标题、内容）
- 按时间范围筛选
- 排序支持（发布时间、浏览量、评论数）
- 分页展示

#### 2.2.5 文章详情
- 文章内容展示
- 浏览量统计（防刷机制）
- 上一篇/下一篇导航
- 相关文章推荐
- 分享功能

### 2.3 分类管理模块

#### 2.3.1 分类创建
- 分类名称
- 分类描述
- 自动生成 Slug（URL 友好标识）
- 支持多级分类（父子关系）

#### 2.3.2 分类编辑
- 修改分类信息
- 调整父分类（移动分类）

#### 2.3.3 分类删除
- 检查分类下是否有文章
- 删除确认
- 可选：将文章移至默认分类

#### 2.3.4 分类查询
- 分类列表（树形结构）
- 分类下文章数量统计

### 2.4 标签管理模块

#### 2.4.1 标签创建
- 自动创建（发布文章时输入新标签）
- 手动创建
- 自动生成 Slug

#### 2.4.2 标签编辑
- 修改标签名称
- 合并标签（可选）

#### 2.4.3 标签删除
- 检查标签关联文章
- 删除确认

#### 2.4.4 标签查询
- 标签列表
- 热门标签排行
- 标签云展示

### 2.5 评论管理模块

#### 2.5.1 评论发布
- 支持游客评论（填写昵称、邮箱、网址）
- 登录用户评论（自动关联用户信息）
- 支持回复评论（嵌套层级）
- 评论内容过滤（敏感词过滤）
- 验证码防刷（可选）

#### 2.5.2 评论审核
- 新评论待审核机制
- 审核通过/拒绝
- 白名单用户免审核

#### 2.5.3 评论管理
- 删除评论
- 标记为垃圾评论
- 批量操作

#### 2.5.4 评论查询
- 文章下评论列表（树形展示）
- 用户评论历史
- 待审核评论列表

### 2.6 首页与展示模块

#### 2.6.1 首页
- 文章列表（分页）
- 轮播图/置顶文章（可选）
- 侧边栏：分类、标签云、热门文章、最新评论

#### 2.6.2 文章归档
- 按年月归档
- 文章数量统计

#### 2.6.3 RSS 订阅
- 文章 RSS 输出

### 2.7 系统管理模块（管理员）

#### 2.7.1 用户管理
- 用户列表查询
- 禁用/启用用户
- 重置用户密码
- 修改用户角色

#### 2.7.2 内容审核
- 待审核文章列表
- 待审核评论列表

#### 2.7.3 系统配置
- 站点基本信息配置
- 评论设置
- SEO 配置

## 3. 数据库设计

### 3.1 实体关系

```
User (1) ----< (N) Post
Category (1) ----< (N) Post
Post (N) ----< (N) Tag (通过 post_tag 关联)
Post (1) ----< (N) Comment
User (1) ----< (N) Comment
Comment (1) ----< (N) Comment (自关联，回复)
```

## 4. 接口设计

### 4.1 RESTful API 规范

- 基础路径：`/api/v1`
- 资源命名使用复数形式
- HTTP 方法对应操作：GET（查询）、POST（创建）、PUT（更新）、DELETE（删除）
- 统一响应格式：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

### 4.2 主要接口列表

#### 用户相关
- `POST /api/v1/auth/register` - 用户注册
- `POST /api/v1/auth/login` - 用户登录
- `POST /api/v1/auth/logout` - 用户登出
- `GET /api/v1/users/profile` - 获取当前用户信息
- `PUT /api/v1/users/profile` - 更新用户信息
- `PUT /api/v1/users/password` - 修改密码

#### 文章相关
- `GET /api/v1/posts` - 文章列表
- `GET /api/v1/posts/{id}` - 文章详情
- `POST /api/v1/posts` - 创建文章（需登录）
- `PUT /api/v1/posts/{id}` - 更新文章（需登录，作者或管理员）
- `DELETE /api/v1/posts/{id}` - 删除文章（需登录，作者或管理员）
- `GET /api/v1/posts/archive` - 文章归档

#### 分类相关
- `GET /api/v1/categories` - 分类列表
- `GET /api/v1/categories/{id}/posts` - 分类下文章列表
- `POST /api/v1/categories` - 创建分类（管理员）
- `PUT /api/v1/categories/{id}` - 更新分类（管理员）
- `DELETE /api/v1/categories/{id}` - 删除分类（管理员）

#### 标签相关
- `GET /api/v1/tags` - 标签列表
- `GET /api/v1/tags/{id}/posts` - 标签下文章列表
- `POST /api/v1/tags` - 创建标签（管理员）
- `PUT /api/v1/tags/{id}` - 更新标签（管理员）
- `DELETE /api/v1/tags/{id}` - 删除标签（管理员）

#### 评论相关
- `GET /api/v1/posts/{id}/comments` - 文章评论列表
- `POST /api/v1/posts/{id}/comments` - 发表评论
- `DELETE /api/v1/comments/{id}` - 删除评论（作者或管理员）
