# 功能开发文档更新检查清单

## 📋 使用说明
每次开发新功能或修改现有功能时，复制此清单并填写，完成后提交。

---

## 🔲 通用检查项（每次必查）

### 代码相关
- [ ] 代码编译通过
- [ ] 单元测试通过并覆盖所有分支
- [ ] 代码符合团队规范
- [ ] 方法注释完整（参数、返回值、异常）
- [ ] 类注释说明职责

### API 文档（Swagger）
- [ ] Controller 方法添加 `@Operation` 注解
- [ ] 请求参数添加 `@Parameter` 注解
- [ ] 响应格式添加 `@ApiResponse` 注解
- [ ] DTO 字段添加 `@Schema` 注解
- [ ] 访问 `/swagger-ui.html` 验证文档正确性

### Git 提交
- [ ] 提交信息包含 `[文档更新]` 标记
- [ ] 列出具体更新的文档内容
- [ ] 关联 Issue 编号（如有）

---

## 🔲 按功能类型检查

### A. 新增功能模块
如涉及以下情况，请检查对应项：

#### 后端代码
- [ ] Entity 类创建并添加注释
- [ ] Mapper 接口和 XML 配置
- [ ] Service 层实现
- [ ] Controller 接口实现
- [ ] DTO/VO 对象创建
- [ ] 枚举/常量类定义

#### 文档更新
- [ ] **博客后端设计文档.md** - 第 2 章添加功能模块说明
- [ ] **博客后端设计文档.md** - 第 4 章添加 API 接口列表
- [ ] Swagger 接口文档完整
- [ ] 数据库 ER 图更新（第 3 章）

#### 数据库
- [ ] Flyway 迁移脚本创建（`V{版本}__{描述}.sql`）
- [ ] **博客后端设计文档.md** - 第 3 章更新表结构说明
- [ ] 索引优化记录

---

### B. 修改现有功能

#### 影响评估
- [ ] 是否影响现有 API？→ 更新 Swagger
- [ ] 是否修改数据库结构？→ 更新迁移脚本和设计文档
- [ ] 是否变更业务逻辑？→ 更新设计文档对应章节
- [ ] 是否影响前端？→ 修改前端相关代码

#### 文档更新
- [ ] 在**博客后端设计文档.md**中标记变更（使用 `**变更**` 标识）
- [ ] 更新 Swagger 接口说明
- [ ] 记录破坏性变更（Breaking Changes）

---

### C. 数据库结构变更

#### 必须完成
- [ ] 创建 Flyway 迁移脚本
- [ ] 脚本幂等性测试（可重复执行）
- [ ] **博客后端设计文档.md** - 第 3 章更新：
  - 表结构说明
  - 字段类型和约束
  - 索引设计
  - ER 图关系

#### 可选完成
- [ ] 数据迁移方案
- [ ] 回滚脚本
- [ ] 性能影响评估

---

### D. 技术栈更新

#### 必须完成
- [ ] **技术栈和约束.md** - 开发技术栈章节
- [ ] pom.xml 依赖版本标注
- [ ] 技术选型说明文档（如适用）

#### 可选完成
- [ ] 性能对比数据
- [ ] 迁移指南
- [ ] 培训材料

---

### E. Bug 修复

#### 重要 Bug
- [ ] 记录问题现象
- [ ] 记录根本原因
- [ ] 记录解决方案
- [ ] 更新相关文档（如逻辑变更）

#### 一般 Bug
- [ ] 确保测试用例覆盖
- [ ] 代码注释更新（如逻辑复杂）

---

## 🔲 文档质量自检

### 准确性
- [ ] 文档描述与代码实现一致
- [ ] 示例代码可运行
- [ ] 参数类型和必填项正确
- [ ] 响应示例与实际一致

### 完整性
- [ ] 所有新增功能都有文档说明
- [ ] 所有接口都有 Swagger 注解
- [ ] 所有数据库变更都有记录
- [ ] 所有配置变更都有说明

### 可读性
- [ ] 语句通顺，无错别字
- [ ] 格式统一，层次清晰
- [ ] 图表清晰，标注完整
- [ ] 专业术语准确

## 💡 使用建议

1. **复制使用**：每次开发新功能时复制此清单到新文件（如 `doc/checklist/功能名称_检查清单.md`）
2. **逐步勾选**：开发过程中实时勾选完成情况
3. **随代码提交**：将完成的清单一并提交到 Git
4. **定期回顾**：每月回顾清单，优化不适用的条目

## 🎯 快速参考

**最简检查流程**（适用于小功能）：
1. ✅ Swagger 注解已添加
2. ✅ 设计文档已更新（如新增功能）
3. ✅ 数据库脚本已创建（如有变更）
4. ✅ Git 提交信息包含文档更新说明

**完整检查流程**（适用于大功能）：
按照上述所有适用项逐一检查
