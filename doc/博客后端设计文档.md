# 博客系统功能设计

## 1. 系统概述

博客系统是一个支持多用户的内容管理平台，提供文章发布、分类管理、标签管理、评论互动等核心功能。

系统集成了 Spring AI 框架，通过阿里云千问（Qwen）大模型提供智能化的内容处理能力，包括自动生成 URL 友好的 Slug、智能内容推荐等功能。

### 2.8 AI 辅助功能模块

#### 2.8.1 AI Slug 生成
- **功能描述**：根据文章/分类/标签标题自动生成 URL 友好的 Slug
- **技术实现**：基于 Spring AI + 阿里云千问（Qwen）大模型
- **支持场景**：
  - 文章发布时自动生成 Slug
  - 分类创建时自动生成 Slug
  - 标签创建时自动生成 Slug
- **中英文处理**：
  - 中文标题：AI 自动翻译为英文后生成 Slug
  - 英文标题：直接转换为小写、空格替换为连字符
- **生成策略**：前端不传入 slug 值，完全由后端 AI 自动生成
- **降级策略**：AI 服务不可用时，自动降级为基于时间戳的 Slug 生成
- **重试机制**：支持最多 3 次重试，确保生成成功率

#### 2.8.2 AI 配置管理
- **模型选择**：支持配置不同的千问模型（qwen-turbo、qwen-plus、qwen-max）
- **温度参数**：可调整生成随机性（默认 0.3，适合确定性输出）
- **API Key 管理**：支持环境变量配置，避免硬编码

## 4. 接口设计

### 4.1 RESTful API 规范

- 基础路径：`/api/v1`
- 资源命名使用复数形式
- HTTP 方法对应操作：GET（查询）、POST（创建）、PUT（更新）、DELETE（删除）
- 统一响应格式：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

### 4.2 主要接口列表

#### 用户相关
- `POST /api/v1/auth/register` - 用户注册
- `POST /api/v1/auth/login` - 用户登录
- `POST /api/v1/auth/logout` - 用户登出
- `GET /api/v1/users/profile` - 获取当前用户信息
- `PUT /api/v1/users/profile` - 更新用户信息
- `PUT /api/v1/users/password` - 修改密码

#### 文章相关
- `GET /api/v1/posts` - 文章列表
- `GET /api/v1/posts/{id}` - 文章详情
- `POST /api/v1/posts` - 创建文章（需登录）
- `PUT /api/v1/posts/{id}` - 更新文章（需登录，作者或管理员）
- `DELETE /api/v1/posts/{id}` - 删除文章（需登录，作者或管理员）
- `GET /api/v1/posts/archive` - 文章归档

#### 分类相关
- `GET /api/v1/categories` - 分类列表
- `GET /api/v1/categories/{id}/posts` - 分类下文章列表
- `POST /api/v1/categories` - 创建分类（管理员）
- `PUT /api/v1/categories/{id}` - 更新分类（管理员）
- `DELETE /api/v1/categories/{id}` - 删除分类（管理员）

#### 标签相关
- `GET /api/v1/tags` - 标签列表
- `GET /api/v1/tags/{id}/posts` - 标签下文章列表
- `POST /api/v1/tags` - 创建标签（管理员）
- `PUT /api/v1/tags/{id}` - 更新标签（管理员）
- `DELETE /api/v1/tags/{id}` - 删除标签（管理员）

#### 评论相关
- `GET /api/v1/posts/{id}/comments` - 文章评论列表
- `POST /api/v1/posts/{id}/comments` - 发表评论
- `DELETE /api/v1/comments/{id}` - 删除评论（作者或管理员）

#### AI 相关
- `POST /api/v1/ai/slug` - AI 生成 Slug（需登录）
- `GET /api/v1/ai/status` - 获取 AI 服务状态

## 7. AI 技术栈

### 7.1 Spring AI 集成
- **Spring AI 版本**：1.0.0（GA）
- **集成方式**：使用 Spring AI OpenAI Starter + 阿里云 DashScope OpenAI 兼容接口
- **模型提供商**：阿里云 DashScope（千问 Qwen）

### 7.2 依赖配置
```xml
<!-- Spring AI BOM -->
<dependency>
    <groupId>org.springframework.ai</groupId>
    <artifactId>spring-ai-bom</artifactId>
    <version>1.0.0</version>
    <type>pom</type>
    <scope>import</scope>
</dependency>

<!-- Spring AI OpenAI Starter（兼容 DashScope） -->
<dependency>
    <groupId>org.springframework.ai</groupId>
    <artifactId>spring-ai-starter-model-openai</artifactId>
</dependency>
```

### 7.3 配置示例
```yaml
spring:
  ai:
    openai:
      # 使用阿里云 DashScope OpenAI 兼容接口
      api-key: ${AI_DASHSCOPE_API_KEY:your-api-key-here}
      base-url: https://dashscope.aliyuncs.com/compatible-mode/v1
      chat:
        options:
          model: qwen-turbo  # 支持 qwen-turbo、qwen-plus、qwen-max
          temperature: 0.3
```

### 7.4 核心服务
- **AiSlugService**：AI Slug 生成服务接口
- **AiSlugServiceImpl**：AI Slug 生成服务实现，支持中英文标题转换

### 7.5 使用示例
```java
@Autowired
private AiSlugService aiSlugService;

public void createPost(CreatePostRequest request) {
    String slug = aiSlugService.generateSlug(request.getTitle());
    post.setSlug(slug);
    // ... 保存文章
}
```

### 7.6 获取 API Key
1. 访问 [阿里云百炼平台](https://www.aliyun.com/product/bailian)
2. 开通 DashScope 服务
3. 在控制台创建 API Key
4. 设置环境变量：`AI_DASHSCOPE_API_KEY=your-api-key`
